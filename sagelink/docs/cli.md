# CLI Reference

The SageLink CLI (`sagelink/cli/sagelink.sage`) provides three subcommands for key management, listening, and connecting.

## Usage

```bash
sage sagelink/cli/sagelink.sage <command> [options]
```

## Commands

### `keygen`

Generate a static X25519 keypair for identity.

```bash
sage sagelink/cli/sagelink.sage keygen
```

Creates two files:
- **`identity.key`** — private key (mode 0600), base64-encoded
- **`identity.pub`** — public key, base64-encoded

### `listen [port]`

Listen for incoming connections, perform Noise_IK handshake, authenticate against pinned peers, and dispatch CMD/FILE/SHELL streams.

```bash
sage sagelink/cli/sagelink.sage listen     # default port 7420
sage sagelink/cli/sagelink.sage listen 9999
```

- Authenticates each connecting peer against `peers.toml` by comparing the static key revealed in msg1
- Spawns a new thread per connection
- Dispatches streams to `handle_cmd_stream`, `handle_file_stream`, or `handle_shell_stream`

### `connect <peer_name> [mode] [args...]`

Connect to a remote peer and initiate a session mode.

```bash
# Interactive shell (default)
sage sagelink/cli/sagelink.sage connect pi4 shell

# Remote command execution
sage sagelink/cli/sagelink.sage connect pi4 cmd "uname -a"

# File transfer (local -> remote)
sage sagelink/cli/sagelink.sage connect pi4 file_send ./local.txt /remote/path.txt
```

**Modes:**

| Mode | Args | Description |
|---|---|---|
| `shell` (default) | — | Interactive PTY shell session |
| `cmd` | `<command>` | Run a command and print output |
| `file_send` | `<local> <remote>` | Transfer file with SHA-256 verification |

## Configuration Files

### `peers.toml`

TOML-format file listing known peers with their public keys and addresses.

```toml
[orangepi]
pubkey = "xT4bP0kM9z2R7wL5qA8cV3nY6sD1fG0hJ2kL4mO..."
addr = "192.168.1.50:7420"

[pi4]
pubkey = "aB3cD5eF7gH9iJ1kL2mN4oP6qR8sT0uV2wX4yZ..."
addr = "192.168.1.51:7420"
```

### `identity.key` / `identity.pub`

Generated by `keygen`. The private key is stored base64-encoded with restricted permissions.

## Implementation Notes

- The CLI parses `peers.toml` with a built-in parser (no external TOML library dependency)
- Key serialization uses a built-in base64 encoder/decoder
- The `connect` subcommand sets `rekey_threshold = 1000` to trigger periodic rekeying
- Connection mode is determined by the 4th positional argument
- The server listener dispatches each connection in a forked thread via `thread.spawn`
